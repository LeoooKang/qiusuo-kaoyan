# 压力测试说明

**Phase**: 1
**测试哲学**: 越残酷越好，找到项目的极限和边界

---

## 测试范围

本次 Phase 1 的测试目标是验证「舰长舰队」实施方案在以下极端场景下的可行性：
- 不同代际舰长对同一知识点的观点冲突
- 零基础学弟学妹的 Fork-PR 门槛
- 隐私/版权内容的误提交风险
- Org 长期无人维护的可持续性

## 测试策略

| 类型 | 是否覆盖 | 测试要点 |
|------|----------|----------|
| 正常输入 | ✓ | 标准 Fork → PR → Merge 流程是否清晰 |
| 边界值 | ✓ | Gen-1 到 Gen-10 的代际命名是否可持续 |
| 异常输入 | ✓ | 如果某人 PR 中误含 PDF/隐私信息，如何拦截 |
| 压力测试 | - | 暂不覆盖（无实际并发场景） |
| 极限测试 | ✓ | 第一代舰长毕业后完全不活跃，母舰能否存活 |
| 破坏性 | ✓ | 两位 Captain 对核心知识点发生激烈争执 |
| 安全 | ✓ | 陌生人恶意 Fork 后删除所有 frontmatter 并发 PR |
