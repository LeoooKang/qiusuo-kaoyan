## 🛠️ 技术偏好

- **平台**: GitHub + Obsidian，不引入额外的复杂基础设施。
- **协议**: 倾向 `CC BY-NC-SA 4.0`（署名-非商业-相同方式共享），强制回馈社区。
- **仓库策略**: Public 核心知识库 + Private 个人日志，物理隔离。
- **组织形式**: GitHub Organization（舰长联盟），而非个人仓库。
- **代际传承**: Fork → 本地优化 → Pull Request，标准的开源协作流。

---

## 🎯 项目偏好

- **愿景优先**：不只是上传笔记，而是打造一个**可传承的知识生态**。
- **隐私零容忍**：任何可能暴露个人信息的内容必须排除。
- **版权合规**：名师 PDF、盗版讲义绝不进入公开仓库。
- **可扩展性**：今天的决策要为未来 3-5 代舰长留有余地。
- **压力测试**：认同 SOP 中「越残酷越好」的测试哲学，愿意主动暴露系统边界。

## ⚡ 快速决策原则

| 场景 | 默认选择 |
|------|----------|
| 简洁 vs 完整 | 在首次沟通时倾向完整，确认后进入简洁执行模式 |
| 成熟方案 vs 创新方案 | 优先成熟方案，除非创新方案能显著增强传承感 |
| 单仓库 vs 多仓库 | 多仓库（Org 生态），但核心仓库必须清晰 |
| Public vs Private | 知识 Public，隐私 Private，不妥协 |
