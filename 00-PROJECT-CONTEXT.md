## 🎯 目标

### 主要目标
1. 将个人 Obsidian 考研笔记库转化为一个**可持续迭代的开源知识母舰**。
2. 建立清晰的**代际传承机制**：第一代舰长 → Fork → 第二代舰长优化 → PR 反哺母舰。
3. 保护个人知识产权，同时最大化知识的社会价值。

> **命名由来**：「路漫漫其修远兮，吾将上下而求索。」求索号是一艘渡所有考研学子上岸的船。

### 次要目标
1. 沉淀一套**考研笔记开源协作的 SOP**，可被其他学科/领域复制。
2. 通过 GitHub Organization 营造仪式感，增强社区归属感。
3. 为未来的自动化工具（如 Dataview 进化看板、AI 辅助批改）预留扩展接口。

---

## 📐 范围

### 包含
- 数学（高数/线代/概率）知识体系与错题方法论
- 专业课（上交 816 自动控制原理）笔记与真题分析
- 英语语料、方法论、文化补充
- 政治答题方法论与考点精炼
- 考研通用方法论（时间管理、心流、复盘模板）
- Obsidian 模板体系（PZ 系列）
- GitHub Organization + 多仓库生态建设

### 不包含
- **高度个人隐私内容**（报考信息、真实姓名电话、具体学校老师评价等）
- 版权敏感的原始教材 PDF（张宇/李林/余丙森等）
- 非考研相关的通用插件代码（由插件市场自行分发）

### 特殊说明：日记的处理
- **保留并公开**：纯情绪波动、学习状态起伏、心态调整方法的日记/周记，对学弟学妹具有参考价值，经脱敏后保留。
- **物理隔离**：涉及具体报考院校选择焦虑、个人隐私细节、敏感人际关系的日记，移出公开仓库。

## ⚠️ 约束

| 类型 | 约束 | 影响 |
|------|------|------|
| 版权 | 不能上传名师原版 PDF/讲义 | 所有 PDF 必须排除 |
| 隐私 | 公开仓库不能有可识别的个人信息 | 高隐私内容物理隔离，低隐私日记脱敏后保留 |
| GitHub | Public 仓库才能被 Fork | 核心知识库最终必须是 Public |
| 技术 | Leo和学弟学妹主要用 Obsidian | 必须保持 `.obsidian/` 配置可用 |
| 时间 | 舰长在校期间维护精力有限 | 文档必须足够自解释 |

## ✅ 成功标准

1. ✅ 核心知识库成功推送至 GitHub Public 仓库，可被 Fork。

3. ✅ 个人隐私和版权敏感内容零泄漏。
4. ✅ 有至少一个学弟学妹成功 Fork 并在本地 Obsidian 中正常打开。
5. ✅ 建立 GitHub Org（可选但推荐），具备长期运营的基础设施。
